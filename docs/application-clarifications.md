# Уточнения для разделов 4.2 и 4.3

## 4.2. Используемые технологии

### Хранилище данных
- **SQL-хранилище:** PostgreSQL.

### Векторный индекс
- **pgvector / Vector DB:** используем **pgvector** как расширение PostgreSQL (встроенный индекс в основной БД, единый кластер, self-hosted). Эксплуатация — in-database (без отдельной внешней векторной БД).

### Backend / frontend
- **Backend:** Node.js + TypeScript, API-слой на **Next.js 14 (App Router)**; для AI-интеграций допускается отдельный stateless-сервис по HTTP (в прототипе функции интеграций могут быть вынесены).
- **Frontend:** React (web-кабинет) на базе Next.js.

### Очереди / планировщик задач
- **Очередь фоновых задач:** PostgreSQL-очередь на таблице `ai_analysis_queue` (с advisory lock).
- **Планировщик:** в приложении, запуск обработчика по событиям (API-триггер) с ретраями и контролем тайм-аутов.

### Интеграции
- **CRM:** Bitrix24 (импорт лидов, запись рейтингов/рекомендаций/статусов).
- **API-обогащение по ИНН:** основной провайдер — **DaData** (ЕГРЮЛ/ЕГРИП); при необходимости возможна интеграция с дополнительными реестровыми поставщиками (Контур.Фокус/СПАРК — по согласованию).
- **LLM API / embeddings:** OpenAI (LLM + эмбеддинги) и/или внутренний AI-сервис.

### Безопасность / аудит
- Роли и права доступа, журнал действий, логирование шагов пайплайна, лимиты по квотам.

## 4.3. Публикации/патенты/ПО Заявителя

### Патент/заявка
- **Статус:** заявка на патент (компьютерно-реализуемый способ/система) подготовлена к подаче.
- **Номер/дата/наименование:** номер и дата будут присвоены при подаче; рабочее наименование — «Система AI-анализа компаний и рекомендации оборудования».

### ПО для ЭВМ
- **Статус:** регистрация ПО не оформлена.
- **План:** регистрация программного комплекса после стабилизации версии MVP (в течение ближайшего релиза).
